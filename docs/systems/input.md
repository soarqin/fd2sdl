# 原版键表与 SDL 统一帧输入

## 1. 范围与结论

本文件记录 `FD2.EXE` 中已由未 patch LE 映像确认的键值语义，并规定 SDL 重写版的宿主输入模型。

原版使用 BIOS 键盘缓冲和 `INT 16h`：

1. `input_check @0x35834` 比较 BIOS Data Area 的键盘缓冲指针，只检查是否有键。
2. 需要具体按键的 UI 调用 `INT 16h, AH=0x10`，取得扫描码和 ASCII 值。

SDL 版只复用原版键表和上下文映射，不复刻 BIOS 跨帧 FIFO。宿主输入统一采用无缓冲帧事件：

- `src/input.c` 是唯一调用 `SDL_PollEvent` 的模块。
- 每次 `fd2_input_begin_frame()` 先丢弃上一帧所有未消费按键，再读取当前已到达的 `SDL_EVENT_KEY_DOWN`。
- 每个按键事件只在当前宿主帧有效。一次 Enter 不能跨对话页、对话框、菜单或战场状态再次触发。
- `SDL_EVENT_QUIT`、系统信号和控制台中断仍是进程级持久请求，不随帧清除。

## 2. 原版依据

### 2.1 `input_check`

`input_check @0x35834` 读取 BDA `0x041a` 与 `0x041c`。两值不等时返回 `1`，相等时返回 `0`。它不修改 BDA 指针。

该事实只用于解释原版逻辑和定位按键检查点。SDL 版的动画跳过查询只检查当前宿主输入帧，不把跳过键留给后续 UI。

### 2.2 `field_key_read`

`field_key_read @0x36cbc` 的原版流程为：

1. 循环调用 `input_check`；无键时推进调色板、动画和 BIOS tick。
2. 将寄存器块 `DS:0x3a8d` 的 AH 设为 `0x10`。
3. 通过保护模式运行库调用 BIOS `INT 16h`。
4. 返回寄存器块中的扫描码。

`0x36d2d..0x36d4c` 的规范化规则：BIOS 返回 `0xe0` 或扫描码 `0x52` 时改为 `0x1c`；扫描码 `0x53` 时改为 `0x01`。

## 4. 上下文映射

### 4.1 片头和 ANI/AFM

当前帧任意 `KEY_DOWN` 可跳过允许输入的动画。下一次 present 会开始新帧并清除未消费事件，因此跳过键不会确认标题菜单。

### 4.2 标题菜单

- Up：选择上移并回绕。
- Down：选择下移并回绕。
- Enter、Space、数字小键盘 0／Enter：确认。
- Esc、H、P、R 不作为标题操作。

### 4.3 二选一框

- Enter、Space、数字小键盘确认：确认。
- Esc、Delete、数字小键盘小数点：取消。
- Left、Right：切换选项。

### 4.4 战场

- 四方向键移动焦点或地图。
- Enter、Space 和数字小键盘确认进入确认路径。
- F2／Home 进入单位详情。
- F1／Page Up 进入辅助页面。
- Esc、Z、Delete／数字小键盘小数点和数字小键盘 5 属于战场焦点更新分支，不与指令菜单取消混用。

### 4.5 战场指令、目标和系统菜单

这些上下文处理四方向、确认和取消。目标选择取消后返回指令菜单；不能把菜单外战场的 Esc 语义扩散到此处。

### 4.6 四槽手工存档选择器

- Up／Down 选择 slot。
- Enter、Space、数字小键盘确认：确认。
- Esc、Delete、数字小键盘小数点：取消。

## 5. SDL 实现约束

- 所有普通按键必须通过 `fd2_input_begin_frame()` 进入当前帧事件数组。
- 不得在动画、对话、菜单或战场中建立第二套按键队列。
- 不得通过「状态边界手工清队列」补救；跨状态隔离由帧生命周期保证。
- 同一帧可以按到达顺序消费多个事件；终止当前 UI 的动作之后必须停止消费。剩余事件会在下一帧自动丢弃。
- 方向键 repeat 保留；确认和取消 repeat 丢弃，防止长按跨 UI 状态重复触发。
- 延时只检查宿主退出，不收集普通按键；普通按键在下一次 begin-frame 时统一读取。
- CTest 必须验证未消费事件在下一帧消失、确认键 repeat 被丢弃、方向 repeat 保留，以及宿主退出请求持续有效。

## 6. 待人工验证

战场 Esc／Z／数字小键盘 5、F1／Page Up、G 的完整可见业务语义仍需 DOSBox 手工操作证据。它们不改变 SDL 无缓冲帧输入约束。
